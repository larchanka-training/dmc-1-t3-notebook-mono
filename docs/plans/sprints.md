### Engineer #1 - JS Execution Engine (MVP)

**Задача**

Реализовать минимальный execution layer для выполнения кода (не UI, а просто функциональность).

**Необходимо обеспечить**

- Перехват вывода `console.log`
- Перехват ошибок выполнения (runtime errors)
- Возврат результата выполнения в UI

**Результат**

- Работающая функция "run cell" в браузере

### Engineer #2 - Notebook Execution UX

**Задача**

Связать пользовательский интерфейс с execution-слоем.

**Необходимо реализовать**

- Кнопку "Run cell" для запуска выполнения кода
- Состояния ячейки:
  - `idle` (ожидание)
  - `running` (выполняется)
  - `success` (успешно выполнено)
  - `error` (ошибка выполнения)
- Отдельную панель вывода (output panel) для каждой ячейки

**Результат**

- Интерактивный notebook с полным циклом выполнения (interactive notebook flow)

### Engineer #3 - Persistence Layer (Notebook Save/Load)

**Задача**

Реализовать механизм сохранения и восстановления ноутбуков (без эндпоинтов).

**Необходимо поддержать**

- Сохранение ноутбука (save notebook)
- Загрузку ноутбука (load notebook)
- Автосохранение с debounce-логикой (auto-save)
- Базовое версионирование формата (`v0`, простой JSON)

**Результат**

- Ноутбуки сохраняются и восстанавливаются после перезагрузки страницы (notebooks persist across reload)
- Документация модели и схемы (обновленная)

### Engineer #4 - API for Execution + Notebooks

**Задача**

Реализовать backend API для работы с notebook и execution.

**Необходимо добавить**

  - `update` (обновление)

- Пользовательский контекст (user context) с заглушкой авторизации (placeholder auth)

**Результат**

- Рабочий notebook API (usable notebook API)

**Notes**
