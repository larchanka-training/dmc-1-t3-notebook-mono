# Практический проект: Генерация кода с помощью LLM в веб-приложении

## Общая информация

Проект представляет собой веб-приложение, аналог Jupyter Notebook, с возможностью генерации кода с помощью LLM. Приложение позволяет создавать заметки, генерировать код на основе текстового описания и вставлять его в ячейки.

**Сроки:** предпоследний спринт; после его завершения через два спринта должен быть готовый продукт.

**Доменное имя:** необходимо купить домен для красивых сертификатов и адресов. Предложения принимаются до завтрашнего утра, затем голосование и покупка.

## Задачи спринта

### 1. Добавление AI (LLM) - проектирование (Tech Lead)

Нужно спроектировать пайплайн генерации:

- Как будет работать генерация?
- Какой файл нужно получить на выходе?
- Принять важное решение:
  - Будет ли LLM работать только через API, который разворачивает DevOps.
  - Или только в браузере (`web LLM`).
  - Или оба варианта: например, если запрос длиннее 200 символов, использовать API, если 200 символов или меньше, использовать браузерную модель.
- Оформить документацию и положить её в архитектурные документы, чтобы все могли пользоваться.

### 2. Развертывание API (DevOps)

- Развернуть LLM на **Bedrock** в AWS.
- Добавить ключи и авторизацию, либо ограничить доступ только из внутренней сети: API должен быть приватным и не должен быть доступен снаружи.
- Выбрать относительно небольшую модель, не самую дорогую.
- В итоге должен получиться рабочий API через Bedrock.

### 3. Подготовка тестов (Test / QA)

Подготовить тесты для генерации функций. Нужно протестировать:

- Генерацию JavaScript-кода.
- Генерацию класса.
- Генерацию React-компонентов или любых других компонентов.
- Обработку ошибок: пустые ответы, неправильные ответы, тайм-ауты.

Все описать и положить в `ai_test_cases.md`.

### 4. Фронтенд: функционал внутри ячейки (Инженер 1)

Реализовать функционал внутри ячейки, который позволяет:

- Нажимать кнопку "Сгенерировать".
- Вставлять сгенерированный код в **существующую пустую следующую ячейку**, если она есть и пуста.
- Или в **новую ячейку**, если следующей нет или она занята.

**Важно:** настоящего запроса к LLM здесь еще быть не должно, нужна заглушка.

### 5. Фронтенд: настоящий запрос к LLM (Инженер 2 - интеграция)

Сделать настоящий запрос к API или к `web LLM`.

**Промт:** должен делать акцент на генерации **только кода**. Если запрос пользователя не просит генерировать код, его нужно отклонять.

### 6. Фронтенд: Context Builder (Инженер 3)

Собрать контекст из ноутбука для отправки в LLM:

- Функция `contextBuilder`, которая возвращает текст.
- Собирать **только необходимую информацию**, важную для генерации кода.
- Например, глобальные переменные, доступные в глобальной области видимости, нужно добавить в контекст, чтобы LLM знала, что их можно использовать.

### 7. Бэкенд: валидация и извлечение кода (Инженер 4)

Это самая сложная задача. Нужно реализовать пайплайн:

1. Получить запрос из ячейки.
2. Отправить запрос в LLM.
3. Получить ответ LLM.
4. **Извлечь код** из текста: убрать Markdown-разметку и лишние описания.
5. **Проверить синтаксис:** убедиться, что код исполняемый. Можно использовать `esbuild` или запуск через консольную команду.
6. Если код невалидный, вернуть LLM информацию об ошибке и заставить переделать ответ.

### 8. Бэкенд: эндпоинт и проверка запросов (Инженер 5)

Сделать эндпоинт, в который приходит запрос с фронтенда.

- Проверка на **prompt injection**. Можно использовать маленькую LLM.
- Доступ к Bedrock и основной LLM должен быть только из внутренней сети.

**Примечание:** возможно, понадобятся две модели:

- Маленькая - для проверки промптов.
- Большая - для генерации кода.

## Логика вставки сгенерированного кода

- Если следующая ячейка существует и она **пустая**, вставляем в нее.
- Если следующей ячейки нет или она **занята** и уже содержит код, создаем новую ячейку и вставляем туда.

---

## Пример использования

Допустим, у вас есть CSV-файл, например выписка по налогам за год. Вы можете:

1. Экспортировать данные в CSV.
2. Вставить содержимое как текст в ячейку.
3. Написать описание: _"Хочу, чтобы мне написали код, который все это спарсит и посчитает"_.
4. Нажать "Сгенерировать" и получить код.
5. Запустить его и работать с данными дальше.

Это инструмент для обработки данных, не требующий написания кода вручную.

## Технологические заметки

### Web LLM (браузерная модель)

- Библиотека: **WebLLM**.
- Работает в браузерах с поддержкой WebGPU, например в Chrome и Firefox.
- В Safari без WebGPU использует WebAssembly.
- Можно указать модель, контекст и температуру.
- Модель скачивается один раз и затем работает локально.

### Ресурсы с примерами

Это open source-решения, которые работают в браузере:

- **Whisper** - аудио и видео в текст; поддерживает несколько моделей с выбором по размеру и качеству.
- Удаление фона у изображений через WebAssembly или WebGPU.
- **Stable Diffusion** - генерация изображений; можно запускать на M1 MacBook.
- Текст в аудио: есть open source-решения, но им требуется не самое слабое железо.

### Платные сервисы для голоса и текста

- Яндекс TTS.
- ElevenLabs, есть несколько бесплатных генераций в месяц.
- Lovan Labs, одни из лидеров рынка.

## Интеграция фронтенда и бэкенда

- Задачи делаются независимо.
- В конце тот, кто делает последнюю из трех задач, а именно фронтенд-запрос, бэкенд-эндпоинт или контекст, объединяет все вместе.
- Интеграция по сути сводится к вызову функции из фронтенда, которая отправляет запрос в API.

## Примечания преподавателя по проекту

- Проект не коммерческий: мы не продаем его и не зарабатываем на нем, но хотим получить готовый продукт через два спринта.
- Если какая-то задача блокирует другую, можно сначала делать решение без валидации, а потом добавить ее.
- В спринте самое сложное - **валидация и извлечение кода** из задачи 7, но она никого не блокирует.
- Для извлечения кода можно использовать простой `regex`.
- Для проверки синтаксиса можно использовать `esbuild` через консольную команду, например из Python.
- Для своих проектов можно использовать LLM, чтобы быстро пробовать новые идеи и не тратить время на рутину.
